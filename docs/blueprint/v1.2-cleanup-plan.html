Phase 1 · Prove the loop is self-sustaining

Start here

Nothing gets deleted until the loop runs a full cycle on its own. This is the report's “done-when”: no laptop in the path, fresh numbers on the public dashboard.

1

Confirm the backend predict deploy is healthy

A stale model contract (max_wave_height_m) had shipped via auto-deploy. The fix is avg_wave_height_m; a manual deploy is repairing it. Verify a manual invoke scores all 10 ports.

aws lambda invoke --function-name dpl-serverless-pilot-predict \
  --region eu-west-3 --profile dpl /tmp/pred.json && cat /tmp/pred.json
# expect: {"status":"ok","scored":10,"failed":0,...}
2

Watch one automatic hourly cycle

The schedule runs features at :15 then predict at :30. After the next :30, the published file should carry today's date and non-null probabilities.

curl -s https://dz4lgcial54jx.cloudfront.net/predictions.json | head -c 300
# expect: generatedAt = today, predictions != all null
3

Confirm the live dashboard shows them

The Vercel site now reads predictions from CloudFront (the baked fixture is gone). Open the production URL and check the 24-hour forecast renders real values.

Gate Only when all three pass is the loop truly self-sustaining. Do not start Phase 2 before this.

Phase 3 · Reclaim infrastructure

Irreversible · gate first

Do this only after Phase 1 has held for a few days. These actions delete data and cannot be undone.

6

Repoint training output before touching the raw bucket

Today train.py uploads the model to S3_BUCKET_RAW (the dpl-raw backfill bucket), then it's copied to the pilot bucket. Delete the raw bucket without fixing this and the retrain workflow breaks.

Order matters Change the training upload target to the pilot data bucket (or point S3_BUCKET_RAW at it) and verify a retrain — then proceed to step 7.
7

Delete the backfill source bucket

Once the ~54 days of AIS are absorbed into the gold tables and the model is verified, the original Terraform-era bucket is dead weight.

aws s3 rb s3://dpl-raw-861276086413 --force --region eu-west-3 --profile dpl
8

Remove the Terraform infra/ dir

SAM (template.yaml) is the single IaC now. The old Terraform state is stale and confusing to keep alongside it.

git rm -r infra/ && git commit -m "chore: remove dead Terraform infra (SAM is IaC)"

Phase 4 · Hardening & tech-debt

Optional · future

Not blocking. Worth a note so they aren't rediscovered as surprises.

9

Watch the predict package size

It sits at 210 MB of the 250 MB unzipped limit (lightgbm + scipy + numpy + vendored libgomp). If a dependency bump pushes it over, switch PredictFunction to a container image (PackageType: Image, up to 10 GB).

10

Revisit the train/test split

Currently a stratified random split (chosen because the chronological tail had zero positives). It carries mild temporal leakage. Once more history accumulates, move back to a time-based split for an honest forward-looking estimate.

11

Consider managed secret storage

The AISStream key lives in .env and is injected by deploy.sh — clean for a solo project. If the team grows, move it to AWS SSM Parameter Store / Secrets Manager and read it at deploy time.
