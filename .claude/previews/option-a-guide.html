Wire your model's predictions into the live dashboard

Six steps. Each is small. Do them in order — don't skip ahead.

DONE = a stranger opens your Vercel URL, switches ports, and sees a live "24h Forecast" badge driven by your model — not hardcoded.

🏛 The one architectural decision

Your predictions are a separate data source from the dashboard's existing port data. Do NOT merge them into DataContext — that couples two pipelines that change for different reasons. Instead: a small hook fetches predictions.json, a card renders it. Two sources, zero coupling, smallest change.

3
Create the decoupled predictions hook
dashboard-v2/src/hooks/usePredictions.js
NEW FILE · src/hooks/usePredictions.js 
import { useState, useEffect } from 'react';

// Fetches predictions.json — a SEPARATE source from DataContext.
// Shape: { "NLRTM": { prediction, probability, as_of } | null }
export function usePredictions() {
  const [predictions, setPredictions] = useState(null);

  useEffect(() => {
    fetch('/predictions.json')
      .then((r) => (r.ok ? r.json() : Promise.reject(r.status)))
      .then(setPredictions)
      .catch(() => setPredictions({})); // graceful: never white-screen
  }, []);

  return predictions;
}
Why the .catch: if the file is missing, the dashboard shows "no forecast" instead of crashing. Failure isolation — same Lens 3 habit, now on the frontend.
